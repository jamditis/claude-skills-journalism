Vibe coding

Practical strategies for building software effectively with AI coding assistants like Claude Code, Cursor, and Windsurf.

What this skill does

Teaches methodologies for effective AI-assisted software development: planning with AI, version control strategies, testing approaches, debugging techniques, and optimizing your workflow across different AI tools.

Core principles

Plan before coding

Start by working with the AI to write a detailed implementation plan in a markdown file. Review it, delete unnecessary items, and mark complex features as "won't do."

Key insight: Use todo lists so both you and the AI can see what's done and what remains.

Clean slate principle

Begin each feature with a clean git state. When stuck, use git reset --hard HEAD if the AI goes down an unproductive path.

Key insight: Multiple failed attempts create layers of bad code that compound problems.

E2E tests over unit tests

Focus on simulating user behavior—testing features by simulating someone clicking through the site or app. LLMs often break unrelated logic.

Key insight: Tests catch regressions before they compound. Ensure tests pass before moving to the next feature.

Clean re-implementation

When you finally find a working solution after several attempts, reset to a clean state and implement it fresh. Don't keep the accumulated mess.

Key insight: A clean re-implementation of a known-good solution is faster than untangling spaghetti.

Effective bug fixing

1 Copy error messages

Simply copy-pasting error messages is often enough context for the AI to identify and fix issues. Don't overthink it.

2 Analyze before coding

Ask the AI to consider multiple possible causes before jumping to implementation. This prevents chasing the wrong problem.

3 Reset after failures

Start with a clean slate after each unsuccessful fix attempt rather than layering fixes on top of broken code.

4 Switch models when stuck

Try different AI models when one gets stuck on a problem. Add strategic logging to understand opaque bugs.

AI tool optimization

Instruction files

Write detailed instructions for your AI in config files.

# Example files:

cursor.rules

windsurf.rules

CLAUDE.md

Local documentation

Download API documentation to your project folder. AI tools work more accurately with local docs.

Better than relying on training data recall.

Compare outputs

Generate multiple solutions and pick the best one rather than accepting the first output.

Different models excel at different tasks.

Tech stack considerations

Works well

  • • Ruby on Rails
  • • Mature frameworks with 10+ years of patterns
  • • Small, modular files
  • • Service-based architectures

Use with caution

  • • Rust, Elixir (less training data)
  • • Files with thousands of lines
  • • Complex monorepos

Best practice

  • • Clear external APIs
  • • Modular architecture
  • • Consistent conventions
  • • Good documentation
